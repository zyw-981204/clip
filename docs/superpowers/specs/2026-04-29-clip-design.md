# clip — macOS 剪贴板历史面板设计文档

**状态**：spec / pending review
**日期**：2026-04-29
**工作名**：`clip`（独立 macOS app）

## 1. 概述

`clip` 是一个 macOS 后台运行的剪贴板历史面板。监听系统剪贴板变化、保留可搜索的历史，用户通过全局热键弹出面板，键盘选择条目后自动粘贴到当前焦点 app。MVP 仅支持纯文本。

定位：本地工具、单用户、无网络、无云同步。

## 2. 目标 / 非目标

### 2.1 MVP 目标

- 后台监听 `NSPasteboard` 文本变化并持久化到本地 SQLite
- 按 500 条 / 30 天的双上限自动淘汰旧条目（pinned 永不淘汰）
- 全局热键唤起面板（默认 `⌃⌥⌘V`，可配置）
- 子串搜索 + 上下方向键导航 + Enter 直接 paste 到焦点 app
- pinned（钉住）支持，pinned 条目永远置顶
- 隐私：识别 `org.nspasteboard.Concealed/Transient/AutoGenerated` 标记并跳过；应用 bundle ID 黑名单
- 应用首启引导（Accessibility 权限、登录项）
- 菜单栏 LSUIElement 图标（无 Dock 图标）

### 2.2 非目标 / 留给以后

- 图片 / 文件 / RTF 内容（v2）
- 全文索引 FTS5（v2 当 history 规模上来时）
- 本地加密（v2）
- iCloud / 跨机同步（永远不做）
- 历史导出/导入 JSON（v2）
- 公证签名分发（v1.1）

## 3. 设计决策汇总

| # | 决策点 | 选择 | 理由 |
|---|---|---|---|
| 1 | 落地形态 | 独立 macOS app（不并入 deskctl） | 剪贴板需要 NSPasteboard 监听 / 全局热键 / paste 注入，全是 Swift 原生 API；deskctl「shell 是主控」的架构不适配 |
| 2 | MVP 内容类型 | 纯文本 | 覆盖 95% 场景，存储模型简单；图片留 v2 |
| 3 | 唤起方式 | 全局热键弹浮窗，菜单栏作为兜底入口 | 高频操作要键盘流；牺牲首装的 Accessibility 权限引导成本 |
| 4 | 隐私策略 | 尊重 Concealed/Transient/AutoGenerated 标记 + 应用 bundle ID 黑名单 | 平衡安全和复杂度；加密留 v2 |
| 5 | 历史保留 | 500 条 / 30 天 双上限，先到先触发；pinned 不计入 | 既挡膨胀又挡陈年内容 |
| 6 | 搜索 | 子串匹配 + 大小写不敏感（SQLite `LIKE`） | 短文本场景下又快又准；模糊匹配在长文本里误命中多 |
| 7 | Pinned | MVP 就做 | 用户日常确实有想钉住的高频片段 |
| 8 | 存储后端 | SQLite via GRDB.swift | Swift 主流、类型安全、迁移友好、未来可加 FTS5 |
| 9 | 全局热键 | KeyboardShortcuts SPM（Sindre Sorhus） | 模块化、自带配置 UI、社区活跃 |
| 10 | 项目位置 | 新独立仓库 `zyw-981204/clip` | 和 deskctl 完全解耦 |
| 11 | 粘贴语义 | 替换：选中条目永久成为新剪贴板内容 | 简单、匹配主流 |
| 12 | 面板默认位置 | 光标位置（撞边自动贴边） | 视线少跳一次 |
| 13 | 删除 pinned | 二次确认 | pinned 是用户显式标记，避免误删 |
| 14 | 面板期间切 app | 自动关闭面板 | 防止粘贴到非预期目标 |

## 4. 架构

### 4.1 进程模型

单 LSUIElement `.app`（菜单栏 accessory，无 Dock 图标），通过登录项常驻后台。所有逻辑在这一个进程，不引入 daemon。

### 4.2 组件分工

| 组件 | 责任 |
|---|---|
| `PasteboardObserver` | 自适应间隔（0.5s / 1s / 3s）轮询 `NSPasteboard.changeCount`，过滤 → 写入 store |
| `PrivacyFilter` | 检查 Concealed/Transient/AutoGenerated UTI 标记 + 应用黑名单 |
| `HistoryStore` | GRDB 包装的 SQLite，提供 `insertOrPromote / search / togglePin / delete / prune` |
| `HotkeyManager` | KeyboardShortcuts 注册全局热键，触发 `togglePanel` |
| `PanelWindow` | 无边框 `NSPanel`（`.nonactivatingPanel`），光标位置弹出，失焦自动关闭 |
| `PanelView`（SwiftUI） | 搜索框 + 列表 + 键盘导航 |
| `PasteInjector` | 写剪贴板 + 关面板 + CGEvent 合成 ⌘V |
| `BlacklistService` | 维护应用 bundle ID 黑名单 |
| `Preferences*`（SwiftUI） | General / Hotkey / Retention / Privacy / Advanced 五个 tab |
| `OnboardingWindow` | 首启引导：Accessibility 权限 + 登录项 + 热键说明 |

### 4.4 存储位置 / 权限

- DB 文件：`~/Library/Application Support/clip/history.sqlite`
- 文件权限：`0600`（用户独占）
- macOS 权限：
  - **Accessibility**：必须，用于 `CGEvent.post` 合成粘贴键
  - 不沙盒化（LSUIElement 非 sandboxed），方便完整访问 pasteboard

## 5. 数据模型

### 5.1 Schema（migration v1）

```sql
CREATE TABLE items (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  content         TEXT    NOT NULL,
  content_hash    TEXT    NOT NULL,           -- SHA256(trim(content))
  source_bundle_id TEXT,                      -- com.apple.Safari
  source_app_name TEXT,                       -- Safari
  created_at      INTEGER NOT NULL,           -- unix seconds
  pinned          INTEGER NOT NULL DEFAULT 0,
  byte_size       INTEGER NOT NULL,
  truncated       INTEGER NOT NULL DEFAULT 0
);
CREATE INDEX idx_items_created ON items(created_at DESC);
CREATE INDEX idx_items_hash    ON items(content_hash);
CREATE INDEX idx_items_pinned  ON items(pinned, created_at DESC);

CREATE TABLE blacklist (
  bundle_id    TEXT PRIMARY KEY,
  display_name TEXT NOT NULL,
  added_at     INTEGER NOT NULL
);

CREATE TABLE schema_version (version INTEGER PRIMARY KEY);
```

### 5.2 关键约定

- 单表 `items` 同时承载 pinned 和普通条目，靠 `pinned` 列区分
- 去重：`content_hash = SHA256(content.trimmingCharacters(in: .whitespacesAndNewlines))`；同 hash 命中已有行 → `UPDATE created_at = now()` 把它顶上来；不插新行（首尾空白差异不算新条目）
- 单条上限 256KB（按 UTF-8 byte 算），超出截断 + `truncated=1`
- 单次复制 > 5MB 直接跳过（先用 `dataForType(.string)?.count` 判断，不读满）
- 时间字段一律 unix seconds（Int64）

## 6. 数据流

### 6.1 采集路径（后台）

```
DispatchSourceTimer (自适应 0.5–3s, leeway 0.1s)
  ↓
读 NSPasteboard.changeCount
  ↓ 变化时
查 Concealed / Transient / AutoGenerated UTI
  ↓ 都不存在
查 .string 内容；空白 / 全空 → 丢
  ↓
读 NSWorkspace.frontmostApplication.bundleIdentifier
  ↓
查 blacklist 命中 → 丢
  ↓
查我们的 com.zyw.clip.internal-paste UTI → 是 → 丢（防自循环）
  ↓
SHA256(trim(content)) → hash
  ↓
HistoryStore.insertOrPromote：
  SELECT id WHERE content_hash = ?
  存在 → UPDATE created_at = now()
  不存在 → INSERT
  ↓
异步 prune（60s + INSERT 后 1/50 概率触发）：
  DELETE WHERE pinned = 0 AND id NOT IN (top 500 by created_at)
  DELETE WHERE pinned = 0 AND created_at < now() - 30d
```

自适应间隔状态机：
- 最近 60s 有变化 → 0.5s（active）
- 默认 → 1s（normal）
- 5 分钟无变化 → 3s（idle）
- 系统睡眠 / 屏幕锁定 → 暂停

### 6.2 唤起路径（用户驱动）

```
全局热键 ⌃⌥⌘V
  ↓
HotkeyManager 记下当前 frontmost app（粘贴目标）
  ↓
PanelWindow 在光标位置弹出（480×640，左上角对齐光标右下方 12pt 偏移；
  若超出当前屏 visibleFrame 自动夹回）
  panel style: .nonactivatingPanel，不偷焦点
  ↓
SwiftUI PanelView 搜索框获焦
  ↓
搜索 onChange (debounce 100ms) → HistoryStore.search(q, limit=50)
  pinned 永远置顶，下面按 created_at DESC
  ↓
用户选条目（↑↓+Enter / ⌘1–9 / 鼠标双击）
  ↓
PasteInjector.paste(item):
  1. NSPasteboard 写选中内容 + 私有 UTI com.zyw.clip.internal-paste
  2. PanelWindow.close()
  3. 等 50ms 让焦点回稳
  4. CGEvent.post(⌘V down + up) → kCGSessionEventTap
  ↓
PasteboardObserver 下个 tick 看到内部 UTI → 跳过
```

### 6.3 面板期间切 app

`NSWorkspace.didActivateApplicationNotification` 监听；除我们自己外，任何 app 切到前台 → `PanelWindow.close()`。

## 8. 性能预算

**目标**：idle CPU < 0.5%，面板 keyDown → searchField first responder < 50ms，list 填充 < 100ms。

| 维度 | 预算 / 实现 |
|---|---|
| 后台 polling wakeups | 自适应 0.5–3s，leeway 0.1s 让 OS 合并；normal 状态 ~3600/hr |
| Idle CPU | < 0.01%（normal 状态实测预期） |
| 启动时间 | `didFinishLaunching` < 200ms；DB migration 后台 queue |
| 启动后 RSS | < 30MB |
| 搜索延迟（500 条） | sub-millisecond |
| 搜索延迟（5000 条） | < 5ms（v2 真到了再升 FTS5） |
| 面板首次打开 | 预创建 `NSPanel`，热路径 < 100ms |
| 面板再次打开 | < 50ms |
| DB | `DatabasePool` + WAL；读并发，写串行；prune 不在主线程 |

## 9. 错误处理 & 边界 case

### 9.1 权限类

| 场景 | 处理 |
|---|---|
| Accessibility 撤销 | 注入前 `AXIsProcessTrustedWithOptions(nil)`；菜单栏红点；点击修复链接到引导 |
| 热键冲突 | KeyboardShortcuts 自带检测；Preferences 显示警告 |
| SMAppService 注册失败 | Preferences 开关回滚 + 错误文案 + 文档指引手动添加 |

### 9.2 数据完整性

| 场景 | 处理 |
|---|---|
| DB 损坏 | 启动时 `PRAGMA integrity_check`；失败 → rename 为 `.corrupted-<ts>` + 新建空库 + 一次性提示 |
| 磁盘满 / 写失败 | 捕获 INSERT 错误；状态栏警告；该次复制丢弃但不崩 |
| Migration 失败 | 错误对话框，用户选「备份并重置」或「退出」；不静默丢数据 |
| Schema 比 app 新 | 拒绝启动，提示升级 app 或手动清空 |
| time skew | 排序加 `id` tiebreak |

### 9.3 大数据 / 高频

| 场景 | 处理 |
|---|---|
| 单次 > 256KB | 截前 256KB + `truncated=1`；面板加 "(截断)" 标识 |
| 单次 > 5MB | 跳过；用 `dataForType(.string)?.count` 提前判断不读满 |
| 用户连按 ⌘C | changeCount 每 tick 看一次，同 tick 多变化只取最后一个 |
| 搜索 query 含 `% _ \` | escape：`%` → `\%`，配合 `ESCAPE '\\'` |

### 9.4 焦点 / 注入

| 场景 | 处理 |
|---|---|
| 保焦点 | `.nonactivatingPanel` 不激活 app，目标 app 始终 frontmost |
| 面板期间切 app | `didActivateApplicationNotification` → 关面板 |
| 注入被 secure input 拒绝 | 内容已在剪贴板，用户可手动 ⌘V，无数据丢失 |
| 自循环 | 内部 UTI 标记，PasteboardObserver 跳过 |

### 9.5 启动 / 多实例

| 场景 | 处理 |
|---|---|
| 双击 .app 启动二实例 | 启动时 `NSRunningApplication.runningApplications(withBundleIdentifier:)` 检查；已存在 → 置前 + 自身退出 |
| 系统睡眠唤醒 | `didWakeNotification` 重启 timer；丢弃首个 changeCount delta |
| 屏幕锁定 | `screensDidLock` 暂停；`screensDidUnlock` 重建基线（不补抓锁屏期间变化） |

### 10.3 集成测试

抽象 `PasteboardSource` 协议，真实实现包 `NSPasteboard.general`，测试用 `FakePasteboardSource`。
`PasteboardObserverTests` 用 fake 驱动 changeCount 序列：

- 普通变化 → 新增 1 条
- 同内容重复 → bump 不新增
- Concealed UTI → 不变
- 黑名单命中 → 不变
- 自身 UTI（self-write loop）→ 不变
- > 5MB → 跳过
- 256KB–5MB → 截断 + `truncated=1`
- 同 tick 内多变化 → 只记录最后

### 10.4 手动 smoke checklist

写到 `docs/MANUAL_TEST.md`，每次发版前过：

- 全新安装 → 首启引导 → Accessibility 授权回流
- 撤销 Accessibility → 红点 → 修复路径
- 热键打开面板 < 100ms
- 在 Safari 复制 → 面板出现该条目
- 1Password 复制密码 → 面板**不**出现
- 加 1Password 到黑名单 → 旧条仍在但新复制不入库
- ↑↓ + Enter 粘贴到 Safari 地址栏成功
- ⌘1–9 直接粘贴
- ⌘P 切换 pin
- ⌘D 删 pinned 弹二次确认
- 面板期间切 app → 自动关闭
- 系统睡眠 5 分钟 → 唤醒 polling 恢复
- 锁屏 → 解锁 → polling 恢复
- 双击 .app → 二实例不启动
- 复制 5MB 文本 → 不卡 UI；面板加 "(截断)"
- 1 小时 idle → Activity Monitor 该进程 < 0.5% CPU

## 11. 验收标准

实现完成的判定：

1. 所有单元 + 集成测试通过
2. `docs/MANUAL_TEST.md` 的全部 checklist 项通过
3. Activity Monitor 实测 1 小时 idle < 0.5% CPU、RSS < 30MB

## 12. 留待后续 / 已知风险

- **签名 / 公证**：v1 ad-hoc 签名，本机能跑；分发要做 Developer ID 签名 + 公证（v1.1）
- **图片 / 文件支持**：v2，需要重构存储模型容纳 binary
- **加密**：v2，可能用 Keychain 派生密钥 + SQLCipher 或 macOS Data Protection
